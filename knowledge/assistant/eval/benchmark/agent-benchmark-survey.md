## 4. τ-bench 详解（2024.6）

### 4.1 核心特点

| 特点 | 说明 |
|------|------|
| **三方交互** | User ↔ Agent ↔ Tool |
| **真实业务** | 航空预订、电商退货 |
| **多策略支持** | Tool-calling, Act, ReAct |
| **自动错误检测** | 交互轨迹分析 |

### 4.2 评测指标

- **pass¹**: 单次尝试成功率
- **pass^k**: k 次尝试可靠性
- **数据库状态对比**: 比较初始与最终状态

### 4.3 性能基准

| 模型 | 航空领域 | 电商领域 |
|------|----------|----------|
| Claude-3-5-Sonnet | 46% | 69.2% |

## 5. 评测可靠性问题

### 5.1 问题统计

| 问题类型 | 受影响评测集 | 比例 |
|----------|--------------|------|
| 可被捷径绕过 | WebArena, τ-bench | 7/10 |
| 存在不可能任务 | 多个 | 7/10 |
| 答案判定不严谨 | WebArena (1.6-5.2%) | - |
| 性能低估 | OSWorld (28%) | - |
| 单元测试缺陷 | SWE-bench (24-41%) | - |

### 5.2 典型问题

```
τ-bench: "什么都不做的 Agent" 通过了 38% 的任务
WebArena: 答案判定存在 1.6-5.2% 的误判率
OSWorld: 过时网站元素导致 28% 性能低估
```

## 6. 企业级评估框架 (CLEAR)

| 维度 | 说明 | 重要性 |
|------|------|--------|
| **C**ost | 成本（50x 差异） | 企业部署关键 |
| **L**atency | 延迟 | 用户体验 |
| **E**fficacy | 效率 | 资源利用 |
| **A**ssurance | 保障（安全、合规） | 企业级必需 |
| **R**eliability | 可靠性（pass@k） | 生产环境 |

## 7. 能力维度与评测对应

| 能力维度 | 关键问题 | 推荐评测集 |
|----------|----------|------------|
| **GUI 操作** | 能否像人一样操作界面？ | WebArena, OSWorld |
| **工具调用** | 能否正确选择和调用 API？ | ToolBench, τ-bench |
| **编程能力** | 能否生成正确代码？ | SWE-bench, HumanEval |
| **复杂推理** | 能否拆解多步任务？ | GAIA |
| **安全合规** | 是否存在风险行为？ | RiOSWorld, SafetyBench |
| **综合能力** | 多环境泛化能力？ | AgentBench |

## 8. 对 assistant-eval 的启示

### 8.1 评测集设计建议

| 建议 | 参考 | 说明 |
|------|------|------|
| 难度分级 | GAIA Level 1/2/3 | 按复杂度分级 |
| 真实场景 | τ-bench 航空/电商 | 从实际使用抽取 |
| 工具调用 | ToolBench, τ-bench | 验证工具序列 |
| 可靠性评估 | CLEAR 框架 | 成本、延迟、可靠性 |

### 8.2 评测指标建议

| 指标 | 来源 | 说明 |
|------|------|------|
| **pass¹** | τ-bench | 单次成功率 |
| **pass^k** | τ-bench | k 次尝试可靠性 |
| **trajectory_score** | LangChain | 轨迹匹配度 |
| **cost/latency** | CLEAR | 企业级指标 |
