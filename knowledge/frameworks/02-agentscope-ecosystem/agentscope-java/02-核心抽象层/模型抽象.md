
# Model 模型抽象

> agentscope-java 核心抽象层 | 统一 LLM 接入

## 一句话

统一的 LLM 接入层，屏蔽不同 Provider 差异，提供一致的响应式流式接口。

---

## 核心接口

```java
public interface Model {
    // 流式生成（唯一方法）
    Flux<ChatResponse> stream(
        List<Msg> messages,      // AgentScope 消息列表
        List<ToolSchema> tools,  // 可用工具（可选）
        GenerateOptions options  // 生成配置（可选）
    );
    
    // 获取模型名称
    String getModelName();
}
```

---

## 类层次结构

```
Model (interface)
└── ChatModelBase (abstract) - 提供 Tracing 包装
    ├── DashScopeChatModel     // 通义千问
    ├── OpenAIChatModel        // GPT-4 / GPT-3.5
    ├── GeminiChatModel        // Google Gemini
    ├── AnthropicChatModel     // Claude
    └── OllamaChatModel        // 本地模型
```

### ChatModelBase 设计

```java
public abstract class ChatModelBase implements Model {
    
    @Override
    public final Flux<ChatResponse> stream(...) {
        // Tracing 包装
        return TracerRegistry.get()
            .callModel(this, messages, tools, options, 
                () -> doStream(messages, tools, options));
    }
    
    // 子类实现具体逻辑
    protected abstract Flux<ChatResponse> doStream(...);
}
```

## 核心数据结构

### ChatResponse

```java
ChatResponse {
    id: String                      // 响应 ID
    content: List<ContentBlock>     // 内容块（TextBlock/ToolUseBlock/ThinkingBlock）
    usage: ChatUsage                // Token 统计
    metadata: Map<String, Object>   // Provider 元数据
    finishReason: String            // 结束原因
}
```

### ChatUsage

```java
ChatUsage {
    inputTokens: int      // 输入 Token
    outputTokens: int     // 输出 Token
    totalTokens: int      // 总 Token
    time: double          // 耗时（秒）
}
```

### GenerateOptions

```java
GenerateOptions {
    // 连接级配置
    apiKey: String
    baseUrl: String
    modelName: String
    stream: Boolean
    
    // 生成参数
    temperature: Double
    maxTokens: Integer

}
```

## Formatter 机制

### 职责
将 AgentScope 统一的 `Msg` 转换为各 Provider 的请求格式

### 流程
```
AgentScope Msg
     ↓
Formatter (Provider 特定)
     ↓
Provider Request Format
     ↓
API Call
```

### Formatter 类型
| Formatter | Provider |
|-----------|----------|
| DashScopeChatFormatter | 通义千问 |
| OpenAIChatFormatter | OpenAI |
| GeminiChatFormatter | Google Gemini |
| AnthropicChatFormatter | Claude |
| OllamaChatFormatter | Ollama |

## 使用示例

### 创建 Model

```java
// DashScope
Model model = DashScopeChatModel.builder()
    .apiKey(System.getenv("DASHSCOPE_API_KEY"))
    .modelName("qwen-plus")
    .stream(true)
    .build();

// OpenAI
Model model = OpenAIChatModel.builder()
    .apiKey(System.getenv("OPENAI_API_KEY"))
    .modelName("gpt-4")
    .stream(true)
    .build();
```

### 调用 Model

```java
// 流式调用
Flux<ChatResponse> responses = model.stream(
    messages,                    // List<Msg>
    toolkit.getToolSchemas(),   // List<ToolSchema>
    GenerateOptions.builder()
        .temperature(0.7)
        .maxTokens(2000)
        .build()
);

// 消费流
responses
    .doOnNext(resp -> {
        String text = resp.getText();
        if (text != null) {
            System.out.print(text);
        }
    })
    .blockLast();
```

### 带工具调用

```java
List<ToolSchema> tools = List.of(
    ToolSchema.builder()
        .name("get_weather")
        .description("Get weather for a city")
        .parameters(weatherSchema)
        .build()
);

Flux<ChatResponse> responses = model.stream(messages, tools, null);
```

## 响应式设计

### 核心原则
- **非阻塞**: 所有 I/O 操作异步
- **流式**: `Flux<ChatResponse>` 支持流式输出
- **背压**: Reactor 自动处理背压

### 注意事项
```java
// ❌ 错误：在 Agent 逻辑中 block
String text = model.stream(msgs, null, null).blockLast();

// ✅ 正确：保持响应式链
return model.stream(msgs, null, null)
    .map(resp -> processResponse(resp))
    .onErrorResume(e -> Mono.just(fallbackResponse));
```
