# 记忆扩展总览

> agentscope-java 长期记忆扩展集合

## 一句话

提供三种长期记忆实现：Mem0（托管服务）、ReMe（自托管）、AutoContext（自动上下文），实现跨会话的持久化记忆。

## 总体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     长期记忆扩展架构                             │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │                 LongTermMemory 接口                      │    │
│  │   - record(List<Msg>): Mono<Void>                       │    │
│  │   - retrieve(Msg): Mono<String>                         │    │
│  └────────────────────────┬────────────────────────────────┘    │
│                           │                                      │
│          ┌────────────────┼────────────────┐                   │
│          ▼                ▼                ▼                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐         │
│  │   Mem0      │  │   ReMe      │  │  AutoContext    │         │
│  │ (托管服务)  │  │ (自托管)    │  │  (自动上下文)   │         │
│  └─────────────┘  └─────────────┘  └─────────────────┘         │
│         │                │                │                     │
│         ▼                ▼                ▼                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐         │
│  │ Mem0 Cloud  │  │ Vector DB   │  │  LLM + 压缩     │         │
│  │ / Self-host │  │ (Milvus等)  │  │  (无需外部依赖) │         │
│  └─────────────┘  └─────────────┘  └─────────────────┘         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 三种记忆扩展对比

| 特性 | Mem0 | ReMe | AutoContext |
|------|------|------|-------------|
| **部署方式** | 托管 / 自托管 | 自托管 | 无外部依赖 |
| **底层存储** | Mem0 Cloud / 本地 | 向量数据库（Milvus 等） | LLM + 压缩算法 |
| **LLM 推理** | ✅ 自动提取记忆 | ✅ 自动提取 | ✅ 压缩总结 |
| **语义搜索** | ✅ 向量检索 | ✅ 向量检索 | ❌ 无 |
| **多租户隔离** | ✅ agent/user/run | ✅ 元数据过滤 | ✅ 按用户隔离 |
| **自定义元数据** | ✅ | ✅ | ❌ |
| **外部依赖** | Mem0 服务 | 向量数据库 | 无 |
| **适用场景** | 快速集成、生产环境 | 私有化部署 | 轻量级、无依赖 |

## Mem0 扩展

### 核心对象

```java
public class Mem0LongTermMemory implements LongTermMemory {
    private final Mem0Client client;      // Mem0 API 客户端
    private final String agentId;          // Agent 标识
    private final String userId;           // 用户标识
    private final String runId;            // 会话标识
    private final Map<String, Object> metadata;  // 自定义元数据
}
```

### 使用案例

```java
// 托管版 Mem0
Mem0LongTermMemory memory = Mem0LongTermMemory.builder()
    .agentName("Assistant")
    .userId("user_123")
    .apiBaseUrl("https://api.mem0.ai")
    .apiKey(System.getenv("MEM0_API_KEY"))
    .build();

// 自托管版 Mem0
Mem0LongTermMemory selfHosted = Mem0LongTermMemory.builder()
    .agentName("Assistant")
    .userId("user_123")
    .apiBaseUrl("http://localhost:8000")
    .apiType(Mem0ApiType.SELF_HOSTED)
    .build();

// 带自定义元数据
Map<String, Object> metadata = new HashMap<>();
metadata.put("category", "travel");
Mem0LongTermMemory withMetadata = Mem0LongTermMemory.builder()
    .agentName("Assistant")
    .userId("user_123")
    .apiBaseUrl("http://localhost:8000")
    .metadata(metadata)
    .build();

// 与 ReActAgent 集成
ReActAgent agent = ReActAgent.builder()
    .name("Assistant")
    .model(model)
    .longTermMemory(memory)
    .longTermMemoryMode(LongTermMemoryMode.BOTH)
    .build();
```

### 优势与劣势

| 优势 | 说明 |
|------|------|
| **快速集成** | 托管服务，无需部署基础设施 |
| **LLM 推理** | 自动提取和推断记忆内容 |
| **语义搜索** | 向量检索，语义匹配准确 |
| **多租户** | agent/user/run 三级隔离 |

| 劣势 | 说明 |
|------|------|
| **外部依赖** | 依赖 Mem0 服务或自托管实例 |
| **成本** | 托管版需要付费 |
| **数据隐私** | 托管版数据存储在云端 |

## ReMe 扩展

### 核心对象

```java
public class ReMeLongTermMemory implements LongTermMemory {
    private final VectorStore vectorStore;    // 向量存储
    private final ChatModel chatModel;         // LLM 用于记忆提取
    private final String userId;               // 用户标识
}
```

### 使用案例

```java
// 创建向量存储（以 Milvus 为例）
VectorStore vectorStore = MilvusVectorStore.builder()
    .host("localhost")
    .port(19530)
    .collectionName("agent_memory")
    .build();

// 创建 ReMe 记忆
ReMeLongTermMemory memory = ReMeLongTermMemory.builder()
    .vectorStore(vectorStore)
    .chatModel(model)
    .userId("user_123")
    .build();

// 与 ReActAgent 集成
ReActAgent agent = ReActAgent.builder()
    .name("Assistant")
    .model(model)
    .longTermMemory(memory)
    .longTermMemoryMode(LongTermMemoryMode.STATIC_CONTROL)
    .build();
```

### 优势与劣势

| 优势 | 说明 |
|------|------|
| **私有化** | 数据完全在本地，隐私可控 |
| **灵活存储** | 支持多种向量数据库 |
| **LLM 推理** | 自动提取记忆内容 |

| 劣势 | 说明 |
|------|------|
| **部署复杂** | 需要部署向量数据库 |
| **维护成本** | 需要维护向量数据库 |
| **性能依赖** | 依赖向量数据库性能 |

## AutoContext 扩展

### 核心对象

```java
public class AutoContextMemory implements LongTermMemory {
    private final ChatModel chatModel;    // LLM 用于压缩
    private final int maxTokens;          // 最大 token 数
    private final String userId;          // 用户标识
}
```

### 使用案例

```java
// 创建 AutoContext 记忆
AutoContextMemory memory = AutoContextMemory.builder()
    .chatModel(model)
    .maxTokens(4000)
    .userId("user_123")
    .build();

// 与 ReActAgent 集成
ReActAgent agent = ReActAgent.builder()
    .name("Assistant")
    .model(model)
    .longTermMemory(memory)
    .build();
```

### 优势与劣势

| 优势 | 说明 |
|------|------|
| **零依赖** | 无需外部服务或数据库 |
| **轻量级** | 部署简单，启动快 |
| **自动压缩** | LLM 自动总结历史对话 |

| 劣势 | 说明 |
|------|------|
| **无语义搜索** | 无法语义检索历史记忆 |
| **信息损失** | 压缩过程可能丢失细节 |
| **LLM 成本** | 每次压缩都调用 LLM |

## 选择建议

| 场景 | 推荐扩展 |
|------|---------|
| **快速原型** | AutoContext（零依赖） |
| **生产环境（云端）** | Mem0（托管版） |
| **私有化部署** | ReMe 或 Mem0（自托管） |
| **高精度检索** | Mem0 或 ReMe（向量检索） |
| **轻量级应用** | AutoContext（无外部依赖） |
