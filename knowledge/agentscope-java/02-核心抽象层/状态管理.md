# 状态管理

> 02-核心抽象层 | 状态持久化基础设施

## 一句话

提供 Agent 状态的持久化和恢复能力，支持会话级别的状态管理。

## 核心组件

| 组件 | 类型 | 作用 |
|------|------|------|
| State | 接口 | 标记接口，表示可持久化的状态对象 |
| StateModule | 接口 | 提供 saveTo/loadFrom 方法 |
| StatePersistence | record | 配置哪些组件需要管理状态 |
| SessionKey | 接口 | 会话标识符 |
| SimpleSessionKey | record | 简单会话标识符（字符串） |

## State 接口

标记接口，无需实现任何方法：

```java
public interface State {}

// 使用 Record 实现
public record AgentMetaState(
    String id,
    String name,
    String description,
    String systemPrompt
) implements State {}
```

## StateModule 接口

提供状态持久化方法：

```java
public interface StateModule {
    // 保存状态
    void saveTo(Session session, SessionKey sessionKey);
    void saveTo(Session session, String sessionId);

    // 加载状态
    void loadFrom(Session session, SessionKey sessionKey);
    void loadFrom(Session session, String sessionId);

    // 如果存在则加载
    boolean loadIfExists(Session session, SessionKey sessionKey);
    boolean loadIfExists(Session session, String sessionId);
}
```

## StatePersistence 配置

控制 ReActAgent 哪些组件需要状态管理：

```java
// 默认：管理所有组件
StatePersistence.all()

// 不管理任何组件
StatePersistence.none()

// 只管理 Memory
StatePersistence.memoryOnly()

// 自定义
StatePersistence.builder()
    .memoryManaged(true)
    .toolkitManaged(false)
    .planNotebookManaged(false)
    .statefulToolsManaged(false)
    .build()
```

| 字段 | 说明 |
|------|------|
| memoryManaged | 是否管理 Memory 状态 |
| toolkitManaged | 是否管理 Toolkit activeGroups |
| planNotebookManaged | 是否管理 PlanNotebook 状态 |
| statefulToolsManaged | 是否管理有状态 Tool 的状态 |

## SessionKey 设计

支持多租户场景：

```java
// 简单场景
SessionKey key = SimpleSessionKey.of("user_123");

// 多租户场景
public record TenantSessionKey(
    String tenantId,
    String userId,
    String sessionId
) implements SessionKey {}

// 使用
session.save(new TenantSessionKey("t001", "u123", "s456"), "agent_meta", state);
```

## 使用示例

```java
Session session = new JsonSession(Path.of("sessions"));

// 加载状态（如果存在）
agent.loadIfExists(session, "user_123");

// ... 使用 agent ...

// 保存状态
agent.saveTo(session, "user_123");
```
