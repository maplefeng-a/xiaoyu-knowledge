# Session 会话管理

> agentscope-java 核心抽象层 | Agent 状态持久化

## 一句话

统一的会话存储接口，支持 Agent、Memory、Toolkit 等组件的跨重启保存与恢复。

---

## Session 接口

```java
public interface Session {
    // 保存单个状态（全量替换）
    void save(SessionKey sessionKey, String key, State value);
    
    // 保存列表（增量追加或全量替换）
    void save(SessionKey sessionKey, String key, List<? extends State> values);
    
    // 获取单个状态
    <T extends State> Optional<T> get(SessionKey sessionKey, String key, Class<T> type);
    
    // 获取列表
    <T extends State> List<T> getList(SessionKey sessionKey, String key, Class<T> itemType);
    
    // 会话管理
    boolean exists(SessionKey sessionKey);
    void delete(SessionKey sessionKey);
    Set<SessionKey> listSessionKeys();
    void close();
}
```

---

## 实现类

| 实现 | 存储方式 | List 策略 | 适用场景 |
|------|----------|-----------|----------|
| `JsonSession` | 文件系统 (.json/.jsonl) | **增量追加** | 本地持久化 |
| `InMemorySession` | 内存 Map | 全量替换 | 测试/临时 |
| `RedisSession` | Redis | - | 分布式部署 |
| `MysqlSession` | MySQL | - | 企业级 |

## JsonSession 存储结构

```
~/.agentscope/sessions/
└── {sessionId}/
    ├── agent_meta.json           # Agent 元数据（单个状态）
    ├── memory_messages.jsonl     # 消息历史（增量追加，JSONL）
    ├── plan_notebook.json        # 计划笔记本
    └── toolkit_activeGroups.json # 工具组状态
```

### 文件格式

**单个状态 (.json)**
```json
{
  "agentId": "agent_123",
  "name": "Assistant",
  "description": "Helpful assistant",
  "systemPrompt": "You are a helpful assistant."
}
```

**列表 (.jsonl - 增量追加)**
```jsonl
{"role":"user","content":[{"type":"text","text":"Hello"}]}
{"role":"assistant","content":[{"type":"text","text":"Hi!"}]}
```

## State 接口

所有可持久化的对象都实现 `State` 接口：

```java
public interface State {
    // 标记接口，表示可序列化到 Session
}
```

**常见实现：**
- `Msg` - 消息
- `AgentMetaState` - Agent 元数据
- `PlanNotebook` - 计划笔记本
- `ToolGroup` - 工具组配置

## 使用示例

### 创建 Session

```java
// 默认目录：~/.agentscope/sessions
Session session = new JsonSession();

// 自定义目录
Session session = new JsonSession(Path.of("my-sessions"));

// 内存 Session（测试用）
Session session = new InMemorySession();
```

### 保存和加载

```java
SessionKey key = SimpleSessionKey.of("user_123");

// 保存单个状态
AgentMetaState meta = new AgentMetaState("agent_1", "Assistant", "Helpful", "...");
session.save(key, "agent_meta", meta);

// 保存列表（增量追加）
List<Msg> messages = List.of(userMsg, agentMsg);
session.save(key, "memory_messages", messages);

// 加载单个状态
Optional<AgentMetaState> loadedMeta = session.get(key, "agent_meta", AgentMetaState.class);

// 加载列表
List<Msg> loadedMessages = session.getList(key, "memory_messages", Msg.class);
```

### 会话管理

```java
// 检查存在
if (session.exists(key)) {
    // 删除会话
    session.delete(key);
}

// 列出所有会话
Set<SessionKey> allSessions = session.listSessionKeys();

// 关闭资源
session.close();
```

## Agent 集成

### Agent 便捷 API

```java
ReActAgent agent = ReActAgent.builder()
    .name("Assistant")
    .model(model)
    .build();

Session session = new JsonSession();
SessionKey key = SimpleSessionKey.of("user_123");

// 保存 Agent 完整状态
agent.saveTo(session, key);

// 从 Session 恢复 Agent
agent.loadFrom(session, key);
```

### saveTo/loadFrom 内部实现

```java
public void saveTo(Session session, SessionKey key) {
    // 1. 保存 Agent 元数据
    session.save(key, "agent_meta", new AgentMetaState(...));
    
    // 2. 保存 Memory
    session.save(key, "memory_messages", memory.getMessages());
    
    // 3. 保存 Plan
    session.save(key, "plan_notebook", planNotebook);
    
    // 4. 保存 Toolkit 状态
    session.save(key, "toolkit_activeGroups", toolkit.getActiveGroups());
}
```

## 增量追加机制

### JsonSession 的智能追加

```java
// 第一次保存：创建文件，写入 2 条消息
session.save(key, "memory_messages", [msg1, msg2]);

// 第二次保存：只追加新消息（通过 hash 判断）
session.save(key, "memory_messages", [msg1, msg2, msg3, msg4]);
// 文件中：msg1, msg2, msg3, msg4（增量追加，不重复）
```

### ListHashUtil

```java
// 通过 hash 判断哪些是新增元素
List<String> existingHashes = readExistingHashes(file);
List<State> newElements = filterNewElements(values, existingHashes);
appendNewElements(file, newElements);
```

```java
    .password("password")
    .build();
```
