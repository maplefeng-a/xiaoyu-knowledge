# Harness 组件详解 - Bash + 代码执行

> **组件定位**: 自主解决问题的通用工具
> **重要性**: ⭐⭐⭐⭐⭐（最强大的组件）
> **实现难度**: ⭐⭐⭐（需要安全控制）

---

## 🎯 核心价值

### 关键转变

```
从：为每个任务设计专用工具
到：提供一个通用工具（Bash）让智能体自己设计工具
```

### 核心理念

**代码执行 = 给智能体一个图灵完备的工具**

就像给人类一台电脑，让它自己想办法解决问题。

---

## 🤔 为什么需要代码执行？

### 问题 1：工具爆炸

**专用工具方案**：
```
需要：
- csv_analyzer_tool        # 分析 CSV
- json_parser_tool         # 解析 JSON
- image_resizer_tool       # 调整图片大小
- pdf_merger_tool          # 合并 PDF
- excel_calculator_tool    # Excel 计算
- ...（无穷无尽）
```

**问题**：
- ❌ 无法预见所有需求
- ❌ 维护成本极高
- ❌ 用户需要学习每个工具

### 问题 2：无法处理未知任务

**场景**：
```
用户：帮我分析这个自定义格式的文件

智能体：我的工具列表里没有这个格式的解析器
用户：那怎么办？
智能体：没办法...
```

### 解决方案：通用工具

**代码执行方案**：
```
只提供：bash + python
智能体：
1. 先看看文件格式
2. 写一个解析脚本
3. 运行脚本
4. 得到结果
```

## 🏗️ 实现原理

### 基础架构

```
┌─────────────────────────────────────┐
│         智能体（Agent）              │
│  "我需要处理一个 CSV 文件"           │
└──────────────┬──────────────────────┘
               │
               ↓
┌─────────────────────────────────────┐
│      代码生成（Code Generation）    │
│  生成 Python 脚本处理 CSV           │
└──────────────┬──────────────────────┘
               │
               ↓
┌─────────────────────────────────────┐
│      Bash 执行（Execution）         │
│  python process_csv.py              │
└──────────────┬──────────────────────┘
               │
               ↓
┌─────────────────────────────────────┐
│      结果观察（Observation）        │
│  读取输出、分析结果                 │
└─────────────────────────────────────┘
```

```python
    def execute_script(self, script: str, language: str = "python") -> ExecutionResult:
        """
        执行脚本

        Args:
            script: 脚本内容

        """
        pass
```

#### 2. 执行结果

```python
@dataclass
class ExecutionResult:
    """执行结果"""
```

## 💻 实际案例

### 案例 1：数据分析

**任务**：分析销售数据

**智能体工作流程**：
```python
# 步骤 1：查看数据
execute("head sales.csv")
# 输出：
# date,product,amount,region
# 2026-01-01,Widget,100,North
# 2026-01-02,Gadget,150,South

# 步骤 2：写分析脚本
write_file("analyze.py", """
import pandas as pd
import matplotlib.pyplot as plt

# 读取数据
df = pd.read_csv('sales.csv')

# 按地区统计
region_sales = df.groupby('region')['amount'].sum()

# 绘制图表
region_sales.plot(kind='bar')
plt.savefig('sales_by_region.png')

# 输出统计结果
print(region_sales.to_string())
""")

# 步骤 3：执行脚本
result = execute("python analyze.py")
# 输出：
# North    5000
# South    7500
# East     3000
# West     6000

# 步骤 4：生成报告
write_file("report.md", f"""
# 销售数据分析

## 按地区统计
{result.stdout}

## 可视化
![销售图表](sales_by_region.png)
""")
```

### 案例 2：Web 爬虫

**任务**：爬取新闻标题

```python
# 智能体写爬虫
write_file("crawler.py", """
import requests
from bs4 import BeautifulSoup

url = "https://news.example.com"
response = requests.get(url)
soup = BeautifulSoup(response.text, 'html.parser')

headlines = soup.find_all('h2', class_='headline')
for h in headlines:

""")

result = execute("python crawler.py")
headlines = result.stdout.strip().split('\n')

# 保存结果
write_file("headlines.txt", '\n'.join(headlines))
```

### 2. 错误处理

```python
def robust_execute(command: str) -> ExecutionResult:
    """健壮的执行"""
    try:
        result = execute(command, timeout=30)

        if result.exit_code != 0:
            logger.error(f"命令执行失败: {result.stderr}")
            # 尝试修复
            fixed_command = try_fix_command(command, result.stderr)
            return execute(fixed_command)

        return result

    except TimeoutError:
        logger.error("执行超时")
        return ExecutionResult(success=False, error="timeout")

    except Exception as e:
        logger.error(f"执行异常: {e}")
        return ExecutionResult(success=False, error=str(e))
```

### 3. 依赖管理

```python
def ensure_dependencies(packages: List[str]):
    """确保依赖已安装"""
    for package in packages:
        result = execute(f"python -c 'import {package}'")
        if result.exit_code != 0:
            execute(f"pip install {package}")
```

## 🎯 性能优化

### 1. 缓存执行结果

```python
from functools import lru_cache

@lru_cache(maxsize=100)
def cached_execute(command: str) -> ExecutionResult:
    """缓存执行结果"""
    return execute(command)
```

### 2. 异步执行

```python
import asyncio

async def execute_async(command: str) -> ExecutionResult:
    """异步执行"""
    proc = await asyncio.create_subprocess_shell(
        command,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE
    )
    stdout, stderr = await proc.communicate()
    return ExecutionResult(
        stdout=stdout.decode(),
        stderr=stderr.decode(),
        exit_code=proc.returncode
    )
```

### 3. 批量执行

```python
def execute_batch(commands: List[str]) -> List[ExecutionResult]:
    """批量执行命令"""
    results = []
    for cmd in commands:
        result = execute(cmd)
        results.append(result)
    return results
```

## 🔍 调试技巧

### 1. 详细日志

```python
def verbose_execute(command: str) -> ExecutionResult:
    """带详细日志的执行"""
    logger.info(f"执行命令: {command}")
    start_time = time.time()

    result = execute(command)

    duration = time.time() - start_time
    logger.info(f"执行完成: {duration:.2f}秒")
    logger.info(f"退出码: {result.exit_code}")

    if result.stdout:
        logger.info(f"输出: {result.stdout[:100]}...")
    if result.stderr:
        logger.error(f"错误: {result.stderr[:100]}...")

    return result
```

## 🛡️ 安全考虑

### 1. 危险命令过滤

```python
DANGEROUS_PATTERNS = [
    r"rm\s+-rf\s+/",      # 删除根目录
    r"sudo\s+",           # 提权
    r">\s*/etc/",         # 修改系统文件
    r"curl.*\|.*bash",    # 远程执行
]

def check_dangerous(command: str) -> bool:
    """检查危险命令"""
    for pattern in DANGEROUS_PATTERNS:
        if re.search(pattern, command):
            return True
    return False
```

```python
def execute_in_sandbox(command: str) -> ExecutionResult:
```
