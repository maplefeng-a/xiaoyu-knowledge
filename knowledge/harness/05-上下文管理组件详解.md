# Harness 组件详解 - 对抗上下文腐烂（Context Rot）

> **组件定位**: 保持长期性能不衰减
> **重要性**: ⭐⭐⭐⭐（性能优化的关键）
> **实现难度**: ⭐⭐⭐⭐（需要智能压缩策略）

---

## 🎯 核心价值

### 什么是上下文腐烂？

**现象**：
```
上下文 10%：模型：我很聪明，推理很快！
上下文 50%：模型：嗯...让我想想...有点慢
上下文 90%：模型：呃...这个...那个...我想不出来了
```

**原因**：
- 上下文窗口有限（200K tokens）
- 塞得越多，模型越"笨"

---

## 🏗️ 三大解决方案

### 1. 压缩（Compaction）

**时机**：上下文快满了（80%）

**策略**：
```python
def compact_context(context):
    """压缩上下文"""
    # 1. 分析当前上下文
    important_info = extract_important(context)

    # 2. 生成摘要
    summary = summarize(important_info)

    # 3. 清空旧上下文
    context.clear()

    # 4. 注入摘要
    context.inject(summary)

    return context
```

**效果**：
```
之前：100K tokens 对话历史
压缩后：5K tokens 摘要
节省：95K tokens！
```

### 2. 工具调用卸载（Tool Call Offloading）

**问题**：
```python
工具输出：1000 行日志（50K tokens）
但只有前 10 行和后 10 行有用
中间 980 行都是噪音
```

**解决**：
```python
def offload_tool_output(output: str, threshold: int = 10000):
    """卸载工具输出"""
    if len(output) > threshold:
        # 保留首尾
        head = output[:1000]
        tail = output[-1000:]

        # 卸载完整内容
        file_path = write_to_file(output)

        # 返回精简版
        return f"{head}\n... [已卸载到 {file_path}] ...\n{tail}"

    return output
```

### 3. Skills（渐进式披露）

**问题**：
```
启动时加载：
- 100 个工具描述
- 50 个 MCP 服务器
- 30 个 Skill 文档

结果：上下文还没开始工作就塞满了
```

**解决**：
```python
class ProgressiveDisclosure:
    """渐进式披露"""

    def __init__(self):
        self.loaded_tools = set()

    def get_tool(self, tool_name: str):
        """按需加载工具"""
        if tool_name not in self.loaded_tools:
            # 动态加载工具描述
            tool_desc = load_tool_description(tool_name)
            inject_to_context(tool_desc)
            self.loaded_tools.add(tool_name)

        return get_tool(tool_name)
```

# 检查上下文使用率
if calculate_tokens(messages) > 0.8 * MAX_TOKENS:

# 执行命令

## 📊 最佳实践

### 1. 上下文监控

```python
class ContextMonitor:
    """上下文监控器"""

    def check_usage(self):
        """检查使用率"""
        usage = self.calculate_usage()
        if usage > 0.8:
            logger.warning(f"上下文使用率：{usage*100}%，建议压缩")
```

### 2. 优先级管理

```python
# 高优先级：保留
- 当前任务描述
- 最新几轮对话
- 关键决策记录

# 低优先级：可压缩
- 早期探索性对话
- 重复的尝试记录
- 冗长的工具输出
```
