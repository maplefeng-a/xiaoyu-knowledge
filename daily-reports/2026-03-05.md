## 📈 关键收获

### 1. 技能系统（Skill）

```java
// 技能 = 指令 + 资源 + 工具
AgentSkill skill = AgentSkill.builder()
    .name("code_review")
    .skillContent("# Code Review...")
    .addResource("checklist.md", "...")
    .build();

skillBox.registration()
    .skill(skill)
    .tool(new ReviewTool())
    .apply();
```

**Skill vs Tool**：
- Skill：粗粒度，包含指令+资源+工具，需要显式加载
- Tool：细粒度，单一功能函数，默认可用

### 2. 规划系统（Plan）

**状态机**：TODO → IN_PROGRESS → DONE/ABANDONED

**状态转换规则**：
1. 标记 IN_PROGRESS 前，所有前置子任务必须 DONE 或 ABANDONED
2. 同一时间只能有一个子任务 IN_PROGRESS
3. finish_subtask 自动激活下一个子任务

### 5. MsgHub 消息总线（深度）

**核心机制：resetSubscribers()**

```java
private void resetSubscribers() {
    if (enableAutoBroadcast) {
        for (AgentBase agent : participants) {
            // 每个Agent订阅"除了自己以外的所有人"
            List<AgentBase> others = participants.stream()
                .filter(a -> !a.equals(agent))
                .collect(Collectors.toList());
            agent.resetSubscribers(name, others);
        }
    }
}
```

**关键设计：**
- 广播的是 `Msg` 对象（完整内容+元数据）
- 必须使用 `MultiAgentFormatter` 区分发言者
- 纯内存，无持久化；生命周期由业务层/Session 管理

### 6. Pipeline 编排（深度）

| Pipeline | 执行方式 | 返回类型 | 适用场景 |
|----------|---------|---------|---------|
| SequentialPipeline | 顺序链式 | `Mono<Msg>` | 研究→分析→撰写 |
| FanoutPipeline | 并行/顺序 | `Mono<List<Msg>>` | 多语言翻译 |

**核心实现：**

```java
// Sequential：flatMap 链式拼接
Mono<Msg> chain = Mono.justOrEmpty(input);
for (AgentBase agent : agents) {
    chain = chain.flatMap(agent::call);
}

// Fanout：merge 并发执行
List<Mono<Msg>> monos = agents.stream()
    .map(agent -> agent.call(input).subscribeOn(scheduler))
    .toList();
return Flux.merge(monos).collectList();
```

### 8. Tracing 可观测性（深度）

**4个拦截点：**

| 拦截点 | 含义 | Span类型 |
|--------|------|----------|
| callAgent | Agent调用 | agent span |
| callModel | 模型调用 | model span |
| callTool | 工具执行 | tool span |
| callFormat | 消息格式化 | format span |

**Reactor Context 传播：**

```java
// TracerRegistry 全局 Hook
Hooks.onEachOperator(HOOK_KEY, Operators.lift((scannable, subscriber) ->
    new CoreSubscriber<Object>() {
        @Override
        public void onNext(Object o) {
            TracerRegistry.get().runWithContext(
                subscriber.currentContext(),
                () -> { subscriber.onNext(o); return null; }
            );
        }
    }
));
```

**接入方式：**
- Studio 自动接入：初始化时自动注册 TelemetryTracer
- Langfuse 手动接入：配置 OTLP endpoint + headers

### 9. 异步流程实战

**全链路响应式：**

```
Controller → Service → ReActAgent → OpenAIChatModel
   Flux       Flux        Flux          Flux
```

**事件流时序：**

```
客户端              Controller         ReActAgent          LLM
  │──POST /stream─────────────────────────────────────────────►│
  │◄──REASONING token──│◄──────────────────│◄──token1────────│
  │◄──REASONING token──│◄──────────────────│◄──token2────────│
  │◄──TOOL_CALL────────│◄──────────────────│◄──tool_call─────│
  │◄──TOOL_RESULT──────│◄──────────────────│◄──result────────│
  │◄──ANSWER───────────│◄──────────────────│◄──final─────────│
  │◄──METRICS──────────│◄──────────────────│                  │
```

**关键点：**
- 每层都是 Flux/Mono，无线程阻塞
- `flatMap` 保持异步链路
- 背压自动传导，不会 OOM
