## 📈 关键收获

### 1. 技能系统（Skill）

```java
// 技能 = 指令 + 资源 + 工具
AgentSkill skill = AgentSkill.builder()
    .name("code_review")
    .skillContent("# Code Review...")
    .addResource("checklist.md", "...")
    .build();

skillBox.registration()
    .skill(skill)
    .tool(new ReviewTool())
    .apply();
```

**Skill vs Tool**：
- Skill：粗粒度，包含指令+资源+工具，需要显式加载
- Tool：细粒度，单一功能函数，默认可用

### 2. 规划系统（Plan）

**状态机**：TODO → IN_PROGRESS → DONE/ABANDONED

**状态转换规则**：
1. 标记 IN_PROGRESS 前，所有前置子任务必须 DONE 或 ABANDONED
2. 同一时间只能有一个子任务 IN_PROGRESS
3. finish_subtask 自动激活下一个子任务

### 3. RAG系统

| 模式 | 说明 | 实现方式 |
|------|------|----------|
| GENERIC | 自动检索，每个 reasoning 前注入 | Hook |
| AGENTIC | Agent 主动决定何时检索 | Tool |
| NONE | 禁用 RAG | - |

### 4. 中断机制

| 来源 | 说明 |
|------|------|
| USER | 用户主动中断 |
| TOOL | 工具执行逻辑中断 |
| SYSTEM | 系统中断（超时/资源限制） |

### 5. Tracing 系统

**4个拦截点**：callAgent / callModel / callTool / callFormat

**关键设计**：
- Reactor Context 传播 tracing 上下文
- 流式响应聚合后写入 span
- 支持 Studio 自动接入 / Langfuse 手动接入

### 6. MsgHub 自动广播

```java
try (MsgHub hub = MsgHub.builder()
        .participants(alice, bob, charlie)
        .build()) {
    hub.enter().block();
    alice.call().block();  // Bob, Charlie 自动收到
}
```

### 7. Pipeline 编排

| Pipeline | 执行方式 | 适用场景 |
|----------|---------|---------|
| SequentialPipeline | 顺序链式 | 研究→分析→撰写 |
| FanoutPipeline | 并行/顺序 | 多语言翻译 |

### 8. 全链路异步

```
Controller → Service → ReActAgent → OpenAIChatModel
   Flux       Flux        Flux          Flux
```

**各层都是 Flux/Mono，无线程阻塞，事件到达即处理。**
