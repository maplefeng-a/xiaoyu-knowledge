
# 规划系统

> 03-Agent能力层 | 复杂任务的分解与追踪

## 一句话

PlanNotebook 提供计划管理工具，将复杂任务分解为子任务并追踪执行状态。

## 核心组件

| 组件 | 作用 |
|------|------|
| Plan | 计划，包含 name/description/expectedOutcome/subtasks |
| SubTask | 子任务，包含 name/description/expectedOutcome/state |
| PlanNotebook | 计划笔记本，提供 10 个工具函数 |
| PlanToHint | 计划转提示策略 |
| PlanStorage | 历史计划存储 |

## 状态机

### PlanState

| 状态 | 说明 |
|------|------|
| TODO | 待执行 |
| IN_PROGRESS | 执行中 |
| DONE | 已完成 |
| ABANDONED | 已放弃 |

### SubTaskState

| 状态 | 说明 |
|------|------|
| TODO | 待执行 |
| IN_PROGRESS | 执行中 |
| DONE | 已完成 |
| ABANDONED | 已放弃 |

## 10 个工具函数

| 工具 | 作用 |
|------|------|
| create_plan | 创建新计划 |
| update_plan_info | 更新计划信息 |
| revise_current_plan | 添加/修改/删除子任务 |
| update_subtask_state | 更新子任务状态 |
| finish_subtask | 标记子任务完成 |
| view_subtasks | 查看子任务详情 |
| get_subtask_count | 获取子任务数量 |
| finish_plan | 完成/放弃计划 |
| view_historical_plans | 查看历史计划 |
| recover_historical_plan | 恢复历史计划 |

## 使用示例

```java
// 创建 PlanNotebook
PlanNotebook planNotebook = PlanNotebook.builder()
    .maxSubtasks(10)
    .needUserConfirm(true)
    .build();

// 在 Agent 中启用
ReActAgent agent = ReActAgent.builder()
    .name("Assistant")
    .model(model)
    .toolkit(toolkit)
    .planNotebook(planNotebook)
    .build();

// 或使用默认配置
ReActAgent agent = ReActAgent.builder()
    .enablePlan()
    .build();
```

## 子任务状态转换规则

1. 标记为 IN_PROGRESS 前，所有前置子任务必须 DONE 或 ABANDONED
2. 同一时间只能有一个子任务处于 IN_PROGRESS
3. finish_subtask 会自动激活下一个子任务

## 自动提示注入

PlanNotebook 通过 Hook 机制在每个 reasoning 步骤前注入上下文提示：

```java
// 获取当前提示
Mono<Msg> hint = planNotebook.getCurrentHint();
```
