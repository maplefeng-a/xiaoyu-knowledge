
# Pi Embedded Runner 嵌入式运行器

> OpenClaw 核心抽象层 | Agent 生命周期管理核心

## 一句话

OpenClaw 的核心运行器，负责嵌入式 Pi Agent 的完整生命周期管理，包括上下文窗口管理、故障转移、压缩、认证配置轮换等关键功能。

---

## 核心设计

### 设计原则
- **嵌入式运行**: 在主进程中运行，共享上下文
- **故障转移**: 多层级重试机制，Auth Profile 轮换
- **上下文管理**: 动态压缩，窗口守卫
- **使用量累积**: 跟踪 input/output/cache token

## 故障分类

### FailoverReason

| 类型 | 说明 | 处理策略 |
|------|------|----------|
| `AUTH_FAILURE` | 认证失败 | 轮换 Auth Profile |
| `RATE_LIMIT` | 速率限制 | 指数退避 |
| `TIMEOUT` | 超时 | 重试 |
| `CONTEXT_OVERFLOW` | 上下文溢出 | 压缩 |
| `BILLING_ERROR` | 计费错误 | 停止运行 |
| `MODEL_OVERLOADED` | 模型过载 | 故障转移 |

## 与 AgentScope-Java 对比

### 相似之处
- 都有上下文管理机制

```typescript
const result = await runEmbeddedPiAgent({
  sessionKey: "session-123",
  model: "claude-3-5-sonnet",

  const result = await runEmbeddedPiAgent(params);
  return result;
} catch (error) {

  }
}
```

## 内部机制

### 重试循环

```
最大重试次数 = min(160, max(32, 24 + profileCount * 8))

for (attempt = 0; attempt < maxRetries; attempt++) {
  try {
    // 选择 Auth Profile
    auth = resolveAuthProfileOrder()[attempt % profileCount];

    // 运行尝试
    result = await runEmbeddedAttempt(params, auth);

    // 成功
    markAuthProfileGood(auth.id);
    return result;
  } catch (error) {
    // 分类错误
    reason = classifyFailoverReason(error);

    // 不可恢复错误
    if (reason === 'BILLING_ERROR') {
      throw error;
    }

    // 标记失败
    markAuthProfileFailure(auth.id, reason);

    // 退避
    await sleepWithAbort(backoff.next(), abortSignal);
  }
}
```
