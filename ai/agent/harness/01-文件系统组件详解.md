# Harness 组件详解 - 文件系统（Filesystem）

> **组件定位**: 持久化存储 + 上下文管理
> **重要性**: ⭐⭐⭐⭐⭐（最基础的组件）
> **实现难度**: ⭐⭐（相对简单）

---

## 🎯 核心价值

### 三大核心能力

```
1. 工作空间（Workspace）
   └─ 读写数据、代码、文档

2. 增量工作（Incremental Work）
   └─ 跨会话存储中间状态

3. 协作界面（Collaboration Surface）
   └─ 多智能体/人机协作的共享表面
```

---

## 🤔 为什么需要文件系统？

### 问题 1：上下文窗口限制

**场景**：
```
用户：帮我分析这个 100MB 的日志文件

模型：好的，把文件内容发给我
用户：文件太大了，发不了
模型：那我没办法了...
```

**分析**：
- 模型上下文窗口有限（200K tokens ≈ 150MB 文本）
- 大文件无法一次性加载
- 需要分批处理

### 问题 2：无法持久化状态

**场景**：
```
第 1 天：
用户：帮我写一个爬虫
模型：好的，开始写...
[写了 500 行代码]
模型：上下文快满了，我记住这些代码了

第 2 天：
用户：继续完善爬虫
模型：什么爬虫？我没印象（上下文已清空）
```

**分析**：
- 模型没有持久化记忆
- 跨会话无法保留工作进度
- 需要外部存储

### 问题 3：协作困难

**场景**：
```
智能体 A：我负责写前端代码
智能体 B：我负责写后端代码
智能体 C：我负责写测试

问题：如何共享代码？
```

**分析**：
- 多智能体需要协作机制
- 需要共享的工作空间
- 需要版本控制

## 💻 实际案例

### 案例 1：大文件分析

**任务**：分析 100MB 日志文件

**传统方式（失败）**：
```python
# 模型尝试一次性读取整个文件
log_content = read_file("huge.log")  # ❌ 上下文溢出
```

**文件系统方式（成功）**：
```python
# 步骤 1：分批读取
batch_size = 1000
errors = []

for i in range(0, total_lines, batch_size):
    # 读取一批
    batch = read_lines("huge.log", i, i+batch_size)

    # 分析这批
    batch_errors = analyze_errors(batch)
    errors.extend(batch_errors)

    # 写入中间结果
    write_file("temp_errors.txt", "\n".join(errors))

# 步骤 2：汇总分析
all_errors = read_file("temp_errors.txt")
summary = generate_summary(all_errors)
write_file("final_report.md", summary)
```

### 案例 2：代码重构项目

**任务**：重构一个大型项目

**工作流程**：
```python
# 第 1 天
write_file("plan.md", "# 重构计划\n- [ ] 重构用户模块\n- [ ] 重构支付模块")
write_file("refactor/user.py", "# 新的用户模块代码")

# 第 2 天
plan = read_file("plan.md")  # 读取昨天的工作
# 继续重构支付模块
write_file("refactor/payment.py", "# 新的支付模块代码")
update_file("plan.md", "- [x] 重构用户模块\n- [ ] 重构支付模块")

# 第 3 天
plan = read_file("plan.md")
# 查看进度，继续工作
```

### 案例 3：多智能体协作

**任务**：开发一个 Web 应用

**协作方式**：
```python
# 智能体 A：前端工程师
write_file("frontend/index.html", "<html>...</html>")
write_file("frontend/app.js", "// 前端代码")

# 智能体 B：后端工程师
write_file("backend/server.py", "# 后端代码")
write_file("backend/api.py", "# API 接口")

# 智能体 C：测试工程师
frontend_code = read_file("frontend/app.js")
backend_code = read_file("backend/server.py")
write_file("tests/test_all.py", "# 集成测试代码")

# 智能体 A：集成测试
test_code = read_file("tests/test_all.py")
run_tests(test_code)
```

```python
def safe_read_file(path: str) -> Optional[str]:
    """安全的文件读取"""
    try:
        if not exists(path):
            logger.warning(f"文件不存在: {path}")
            return None
        return read_file(path)
    except Exception as e:
        logger.error(f"读取文件失败: {path}, {e}")
        return None
```

## 🎯 性能优化

### 1. 大文件处理

```python
# ❌ 不好：一次性读取
content = read_file("huge.log")  # 内存溢出

# ✅ 好：流式读取
def process_large_file(path: str, batch_size: int = 1000):
    with open(path, 'r') as f:
        batch = []
        for line in f:
            batch.append(line)
            if len(batch) >= batch_size:
                yield batch
                batch = []
        if batch:
            yield batch
```

### 2. 缓存机制

```python
class CachedFileSystem:
    """带缓存的文件系统"""

    def __init__(self):
        self.cache = {}

    def read_file(self, path: str) -> str:
        # 检查缓存
        if path in self.cache:
            return self.cache[path]

        # 读取文件
        content = super().read_file(path)

        # 缓存
        self.cache[path] = content
        return content
```

### 3. 并发控制

```python
from threading import Lock

class ThreadSafeFileSystem:
    """线程安全的文件系统"""

    def __init__(self):
        self.locks = defaultdict(Lock)

    def write_file(self, path: str, content: str):
        with self.locks[path]:
            super().write_file(path, content)
```
