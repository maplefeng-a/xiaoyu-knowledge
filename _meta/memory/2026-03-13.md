## 📚 核心学习：智能体系统协议分层架构

### 一、协议分层模型（关键洞察）

```
应用层（消息格式）
    ↓
传输层（通信协议）
    ↓
智能体运行时（Agent 框架）
    ↓
模型层（LLM API - SSE）
    ↓
大模型服务
```

**核心结论：**
- **模型层 → 运行时：SSE（单向推送）**
- **运行时 → 客户端：WebSocket 或 HTTP + SSE**
- **传输层通用，应用层协议才是重点**

### 二、WebSocket vs SSE 选择原则

| 场景 | 推荐协议 | 原因 |
|------|---------|------|
| 模型推理输出 | SSE | 单向推送，自动重连 |
| 控制平面（Gateway） | WebSocket | 双向交互，RPC 模式 |
| 简单聊天应用 | SSE | 简单可靠 |
| 需要中断/控制 | SSE + HTTP | 混合模式 |

## 🏗️ OpenClaw Gateway 架构

### 核心设计
- **单一控制平面** - 一个 Gateway 统一管理所有连接
- **协议：WebSocket + JSON**
- **角色：operator（控制平面）/ node（能力宿主）**
- **认证：设备签名 + 配对审批**

### Channel Provider 抽象
- **统一消息模型：** InboundMessage / OutboundMessage
- **能力声明：** reactions / edit / delete / threads / typing / media
- **插件式架构：** 实现 Provider 接口即可接入新 Channel
- **健康监控：** 自动重连 + 指数退避

## 📦 AgentScope 协议体系

### Msg 消息格式
```json
{
  "id": "abc123",
  "name": "Assistant",
  "role": "assistant",
  "content": [
    { "type": "text", "text": "Hello" },
    { "type": "tool_use", "id": "tool_1", "name": "search", "input": {...} }
  ],
  "metadata": {...},
  "timestamp": "2026-03-13 ..."
}
```

### ContentBlock 类型
- text / tool_use / tool_result
- image / audio / video
- thinking（Java 特有）

### 协议层次
1. **消息协议** - Msg + ContentBlock
2. **A2A 协议** - 跨智能体通信
3. **MCP 协议** - 工具集成
4. **工具协议** - 工具调用

## 💡 关键洞察

### 1. AgentScope 只定义应用层
- **不定义传输协议**（交给 HTTP 客户端）
- **只定义：** Msg 消息格式、Agent 接口、模型适配器

### 2. OpenClaw 定义了完整运行时
- **应用层：** Gateway Protocol
- **传输层：** WebSocket
- **运行时：** 会话管理、状态持久化
- **Channel 集成：** Provider 抽象

### 3. 传输层选择决策树
```
单向推送 → SSE
双向交互 → WebSocket
请求-响应 → HTTP API
```

### 4. Spring + AgentScope 本质
```
前端 SSE → Spring SSE → AgentScope → 模型 SSE
         （透传）
```
