
# 多智能体协作

> agentscope-java 核心能力 | MsgHub + Pipeline 编排

## 一句话

MsgHub 处理消息广播，Pipeline 处理执行编排，两者组合实现多智能体协作。

## 两大组件

| 组件 | 职责 | 模式 |
|------|------|------|
| MsgHub | 消息分发 | 发布-订阅 |
| Pipeline | 执行编排 | 责任链/扇出 |

## Pipeline 接口

```java
public interface Pipeline<T> {
    Mono<T> execute(Msg input);
    Mono<T> execute(Msg input, Class<?> structuredOutputClass);
    default String getDescription() { ... }
}
```

## SequentialPipeline（顺序执行）

### 执行流程

```
Input → Agent1 → Agent2 → ... → AgentN → Output
```

### 核心实现

```java
@Override
public Mono<Msg> execute(Msg input, Class<?> structuredOutputClass) {
    Mono<Msg> chain = Mono.justOrEmpty(input);

    // 前 N-1 个 Agent 使用普通调用
    for (int i = 0; i < agents.size() - 1; i++) {
        AgentBase agent = agents.get(i);
        chain = chain.flatMap(agent::call);
    }

    // 最后一个 Agent 支持结构化输出
    AgentBase lastAgent = agents.get(agents.size() - 1);
    if (structuredOutputClass != null) {
        chain = chain.flatMap(msg -> lastAgent.call(msg, structuredOutputClass));
    } else {
        chain = chain.flatMap(lastAgent::call);
    }

    return chain;
}
```

### 使用示例

```java
SequentialPipeline pipeline = SequentialPipeline.builder()
    .addAgent(researcher)   // 第一步：研究
    .addAgent(analyst)      // 第二步：分析
    .addAgent(writer)       // 第三步：撰写
    .build();

Msg result = pipeline.execute(input).block();
```

## FanoutPipeline（并行执行）

### 执行流程

```
        ┌─→ Agent1 ─→ Output1
        │
Input ──┼─→ Agent2 ─→ Output2
        │
        └─→ Agent3 ─→ Output3
```

### 核心实现

```java
// 并发执行
private Mono<List<Msg>> executeConcurrent(Msg input, Class<?> structuredOutputClass) {
    List<Mono<Msg>> agentMonos = agents.stream()
        .map(agent -> agent.call(input)
            .subscribeOn(scheduler))  // 每个Agent独立调度
        .toList();

    return Flux.merge(agentMonos).collectList();
}

// 流式执行（支持 Streaming）
public Flux<Event> stream(Msg input, StreamOptions options) {
    List<Flux<Event>> agentFluxes = agents.stream()
        .map(agent -> agent.stream(input, options)
            .subscribeOn(scheduler))
        .toList();

    return Flux.merge(agentFluxes);  // 事件交错合并
}
```

### 使用示例

```java
FanoutPipeline pipeline = FanoutPipeline.builder()
    .addAgent(translatorEN)  // 并行翻译
    .addAgent(translatorJP)
    .addAgent(translatorDE)
    .concurrent()            // 启用并发
    .build();

List<Msg> results = pipeline.execute(input).block();
// results.size() == 3
```

### 并发 vs 顺序

```java
// 并发模式（默认）
FanoutPipeline concurrent = FanoutPipeline.builder()
    .addAgents(agents)
    .concurrent()
    .build();

// 顺序模式
FanoutPipeline sequential = FanoutPipeline.builder()
    .addAgents(agents)
    .sequential()
    .build();
```

### 自定义调度器

```java
FanoutPipeline pipeline = FanoutPipeline.builder()
    .addAgents(agents)
    .concurrent()
    .scheduler(Schedulers.parallel())  // 自定义调度器
    .build();
```

## 协作模式总结

### 模式1：广播讨论（MsgHub）

```java
try (MsgHub hub = MsgHub.builder()
        .participants(alice, bob, charlie)
        .build()) {
    hub.enter().block();
    alice.call().block();  // 其他人自动收到
    bob.call().block();
}
```

### 模式2：链式处理（SequentialPipeline）

```java
SequentialPipeline pipeline = SequentialPipeline.builder()
    .addAgent(researcher)
    .addAgent(analyst)
    .addAgent(writer)
    .build();
Msg result = pipeline.execute(input).block();
```

### 模式3：并行处理（FanoutPipeline）

```java
FanoutPipeline pipeline = FanoutPipeline.builder()
    .addAgents(translators)
    .concurrent()
    .build();
List<Msg> results = pipeline.execute(input).block();
```

### 模式4：混合编排

```java
// 并行翻译 → 顺序审核 → 广播发布
FanoutPipeline translators = ...;
SequentialPipeline reviewers = ...;

List<Msg> translations = translators.execute(input).block();
Msg reviewed = reviewers.execute(merge(translations)).block();

try (MsgHub hub = MsgHub.builder()
        .participants(alice, bob, charlie)
        .announcement(reviewed)
        .build()) {
    hub.enter().block();
}
```
